[draft] Keyboard Input in Fullscreen Safari

December 2014 to my astonishment I found out Safari blocks keyboard inputs in fullscreen mode. Ever since September 2013 it turns out for security reasons it is no longer possible to request keyboard support by calling the webkitRequestFullScreen() method with a flag ALLOW_KEYBOARD_INPUT.

The ability to accept alphanumeric keyboard events in fullscreen mode is considered a security risk by Apple. I don’t really know why. My best guess is that it could allow for websites to mimic system UI and upon entering fullscreen mode to trick users into entering sensitive data.

Safari documentation goes on to explain that the webkitRequestFullscreen() method can only be invoked in response to a user action, such as clicking a button. It cannot be invoked in response to a load event, for example.

To me this solves the security issue. But today the documentation no longer even mentions ALLOW_KEYBOARD_INPUT. It looks The fullscreen functionality is still part of HTML audio and video.

There is a real use case as to why we need a fully capable fullscreen API.

In macOS fullscreen mode was embraced after the success of iPad Apps.

CMS, distractions free solutions, parental control, admin panels… I can think of a number of cases where not only keyboard input, but in fact ability to stay locked into fullscreen can enhance greatly the user experience, and even open new markets.

I’m inviting everyone interested to help, comment, find solutions, and continue the discussion for proper fullscreen support.
